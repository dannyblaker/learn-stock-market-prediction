# Contributing to Stock Market ML

Thank you for your interest in contributing! This document provides guidelines for contributing to this educational repository.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Check existing issues before creating new ones
- Provide clear descriptions and reproducible examples

### Adding New Models or Methods

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/new-model`
3. **Follow the existing structure**:
   - Place code in appropriate `src/` subdirectory
   - Include docstrings with explanations
   - Add example usage in `__main__`
   - Update relevant notebooks

4. **Include**:
   - Clear explanations of the approach
   - When to use it (pros/cons)
   - Working example
   - References to papers/resources

5. **Test your code**
6. **Submit a pull request**

## Code Style

- Follow PEP 8 guidelines
- Use descriptive variable names
- Include type hints where helpful
- Add comments for complex logic
- Keep functions focused and modular

## Documentation

- Write clear docstrings (Google style)
- Explain financial concepts for non-finance developers
- Include usage examples
- Add entries to GLOSSARY.md for new terms

## Examples of Good Contributions

- New ML models with explanations
- Improved feature engineering methods
- Better evaluation metrics
- Tutorial notebooks
- Bug fixes
- Documentation improvements
- Performance optimizations

## What We're Looking For

- Educational value (helps developers learn)
- Clear explanations
- Working code examples
- Practical applicability
- Good documentation

## What to Avoid

- Overly complex code without explanation
- Methods without clear use cases
- Incomplete implementations
- Code without examples
- Missing documentation

## Questions?

Open an issue or reach out to maintainers.

Thank you for helping make financial ML more accessible!
